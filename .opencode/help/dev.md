# /dev - 开发代码

## 描述

根据产品需求文档实现功能代码，自动判断项目类型和技术栈。

## 前置条件

- **必须存在**：Product-Spec.md

## 输入

- `Product-Spec.md` - 产品需求文档
- 现有代码文件（如 package.json 等）

## 输出

- **输出文件**：代码文件（src/ 目录）
- **包含内容**：
  - 功能模块代码
  - 组件代码
  - API 接口
  - 配置文件

## 工作流程

```
用户输入 /dev
  ↓
主控检测 Product-Spec.md 是否存在
  ↓
检测现有项目类型和技术栈
  ↓
确定是新建还是扩展现有项目
  ↓
按照优先级实现功能代码
  ↓
对照产品文档检查完整度
```

## 技术栈选择

### 前端技术栈

| 框架 | 特点 | 适用场景 |
|------|------|----------|
| React/Next.js | 生态丰富、SSR支持 | Web应用、企业级应用 |
| Vue 3/Nuxt.js | 简单易学、渐进式 | 中小型项目 |
| React Native | 跨平台开发 | 移动应用 |

### 后端技术栈

| 框架 | 特点 | 适用场景 |
|------|------|----------|
| Node.js/Express | 高性能、异步I/O | API服务、微服务 |
| Python/FastAPI | 简洁、高效、AI集成 | 数据处理、AI服务 |
| Go | 高并发、编译型 | 高性能服务 |

### 数据库

| 数据库 | 特点 | 适用场景 |
|--------|------|----------|
| PostgreSQL | 功能强大、支持JSON | 关系型数据 |
| MongoDB | 灵活、文档型 | 内容管理、日志 |
| Redis | 高速、缓存 | 缓存、Session |

## 功能实现顺序

1. **高优先级**：核心功能，必须实现
2. **中优先级**：重要功能，建议实现
3. **低优先级**：锦上添花，可选实现

## 相关指令

| 指令 | 描述 | 关系 |
|------|------|------|
| `/confirm` | 设计确认 | 前置步骤 |
| `/run` | 本地运行 | 下一步 |
| `/test` | 单元测试 | 后续步骤 |

## 后续操作

代码开发后，建议：
1. 使用 `/run` 验证功能是否正常
2. 使用 `/test` 执行单元测试
3. 使用 `/review` 进行代码审查
