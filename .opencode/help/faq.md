# 常见问题（FAQ）

## Q1: 如何开始一个新项目？

**答**：输入 `/prd` 开始需求收集。产品经理会通过追问帮助你明确需求，然后生成 Product-Spec.md。

## Q2: 可以跳过某些阶段吗？

**答**：可以，但建议按流程走。如果跳过某个阶段，可能会有以下问题：
- 跳过 /prd：需求不清晰，可能返工
- 跳过 /ui：UI 提示词不完整
- 跳过 /confirm：设计不符合预期
- 跳过 /test：代码质量无法保证
- 跳过 /review：可能存在潜在问题

## Q3: 迭代开发时如何添加新功能？

**答**：
1. 输入 `/prd`（迭代模式）
2. 产品经理会更新 Product-Spec.md
3. 然后使用 /dev 实现新功能
4. 后续流程同上

## Q4: /art 指令如何使用？

**答**：输入 `/art` 后，系统会检测可用的设计生成方式：
1. **canvas-design**：快速生成，无需配置
2. **MCP 服务**：如 Nano Banana（需要配置）

选择后，系统会生成高保真设计稿到 design-assets/ 目录。

## Q5: 测试失败怎么办？

**答**：
1. 查看 test-report.md 了解失败原因
2. 修复代码中的问题
3. 重新运行 `/test`

## Q6: 代码审查未通过怎么办？

**答**：
1. 查看 code-review-report.md
2. 重点关注 Critical 和 Major 问题
3. 修复后重新运行 `/review`

## Q7: 部署失败怎么办？

**答**：
1. 查看 deployment-report.md
2. 检查 CI/CD 配置
3. 检查云平台配额和权限
4. 查看部署日志定位问题

## Q8: 如何回退到之前的阶段？

**答**：工作流是单向的，如果需要回退：
1. 删除或修改对应的产物文件
2. 重新运行相应的指令

例如要重新生成设计稿：
```bash
rm -rf design-assets/
/art
```

## Q9: 如何查看项目当前状态？

**答**：输入 `/status` 查看：
- 当前模式（0-1 或迭代）
- 已完成的阶段
- 待办事项

## Q10: 可以自定义帮助内容吗？

**答**：可以。在 `.opencode/help/` 目录下添加自定义帮助文件：
- `menu.md` - 自定义主菜单
- `prd.md` - /prd 指令帮助
- `ui.md` - /ui 指令帮助
- `faq.md` - 常见问题

如果自定义文件存在，会优先使用。
