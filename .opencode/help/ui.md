# /ui - 生成 UI 提示词

## 描述

根据产品需求文档自动生成原型图提示词，包含视觉风格、配色方案和界面元素描述。

## 前置条件

- **必须存在**：Product-Spec.md

## 输入

- `Product-Spec.md` - 产品需求文档

## 输出

- **输出文件**：`UI-Prompts.md`
- **包含内容**：
  - 视觉风格建议
  - 配色方案
  - 界面元素描述
  - 多个设计方案

## 工作流程

```
用户输入 /ui
  ↓
主控检测 Product-Spec.md 是否存在
  ↓
读取产品文档，提取核心功能
  ↓
确定设计风格和配色方案
  ↓
为每个核心功能生成提示词
  ↓
生成 UI-Prompts.md
```

## 视觉风格选项

| 风格 | 特点 | 适用场景 |
|------|------|----------|
| Modern Minimalist | 干净、简洁、留白多 | Web应用、工具类 |
| Material Design | 卡片式、阴影效果、动效丰富 | Android应用、Web |
| Neumorphism | 软阴影、立体感、柔和 | 创意工具、智能家居 |
| iOS Human Interface | 毛玻璃、圆角、动画流畅 | iOS应用、高端产品 |
| Flat Design | 无阴影、纯色块、图标化 | 移动应用、简洁工具 |

## 配色方案

根据行业特性选择：

| 行业 | 推荐配色 |
|------|----------|
| 科技类 | 蓝色系、深色模式、霓虹色 |
| 金融类 | 深蓝、金色、灰色 |
| 教育类 | 暖色调、绿色、蓝色 |
| 娱乐类 | 鲜艳色、渐变色、活泼色 |
| 工具类 | 中性色、简洁色、专业色 |

## 相关指令

| 指令 | 描述 | 关系 |
|------|------|------|
| `/prd` | 需求收集 | 前置步骤 |
| `/art` | 生成设计稿 | 下一步 |
| `/confirm` | 确认设计 | 后续步骤 |

## 后续操作

UI 提示词生成后，建议：
1. 使用 `/art` 生成设计稿
2. 使用 `/confirm` 确认设计方向
