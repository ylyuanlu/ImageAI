# /art - 设计稿生成

## 描述

根据 UI 提示词生成高保真设计稿，支持多种生成方式（canvas-design 或 MCP 服务）。

## 前置条件

- **必须存在**：UI-Prompts.md
- **输入文件**：UI-Prompts.md（包含视觉风格、配色方案、界面元素描述）

## 输出

- **输出目录**：`design-assets/`
- **输出文件**：
  - PNG/PDF 设计稿
  - design-philosophy.md（设计哲学文档，可选）

## 使用场景

- 完成 UI 提示词生成后
- 需要将提示词转化为可视化设计
- 需要生成高保真 UI 稿供开发参考

## 工作流程

```
用户输入 /art
  ↓
主控检测 UI-Prompts.md 是否存在（必须存在）
  ↓
检测可用的设计生成方式
  ↓
（如果存在多个选项）让用户选择生成方式
  ↓
调用相应的技能生成设计稿
  ↓
生成 design-assets/ 目录（含设计稿文件）
  ↓
流程结束
```

## 生成方式选择

系统会检测可用的设计生成方式：

### 方式 1：canvas-design（默认）

**优点**：
- 快速生成，无需配置
- 生成设计哲学 + PNG/PDF
- 完全本地化，不依赖外部服务

**适用场景**：
- 快速原型设计
- 需要设计理念说明
- 无外部服务配置

### 方式 2：MCP 服务（如 Nano Banana）

**优点**：
- 生成高保真 AI 设计稿
- 支持更多样式选项
- 可调整参数（风格、细节程度等）

**配置要求**：
MCP 服务需要在配置中声明 image-generation tag：
```json
{
  "mcpServers": {
    "nano-banana": {
      "command": "...",
      "args": [...],
      "tags": ["image-generation", "design"]
    }
  }
}
```

**适用场景**：
- 需要更高质量的设计稿
- 有特定的设计风格要求
- 需要更多参数调整

## 相关指令

| 指令 | 描述 | 关系 |
|------|------|------|
| `/ui` | 生成 UI 提示词 | 前置步骤 |
| `/confirm` | 确认设计稿 | 下一步 |
| `/dev` | 开始代码开发 | 后续步骤 |

## 后续操作

设计稿生成后，建议：
1. 使用 `/confirm` 确认设计方向
2. 如果需要修改，使用 `/art` 重新生成
3. 确认后使用 `/dev` 开始代码开发
