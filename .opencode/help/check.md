# /check - 完整度检查

## 描述

对照 Product-Spec.md 检查功能是否完整，列出已实现和未实现功能。

## 前置条件

- **Product-Spec.md 必须存在**

## 参考学习

使用全局技能 `verification-before-completion` 学习完成验证最佳实践

## 工作流程

```
用户输入 /check
  ↓
主控读取 Product-Spec.md
  ↓
对照检查功能完整度
  ↓
列出已实现功能
  ↓
列出未实现功能
  ↓
提出补充建议
```

## 检查内容

| 检查项 | 描述 |
|--------|------|
| 核心功能 | 产品文档中的核心功能是否都已实现 |
| 功能优先级 | 高优先级功能是否完整 |
| 业务规则 | 产品文档中的业务规则是否正确实现 |
| 输入验证 | 输入验证是否完整 |
| 错误处理 | 异常处理是否完善 |
| UI 一致性 | 界面是否符合设计稿 |

## 后续操作

检查完成后：
1. **完整度 > 90%**：可进行发布
2. **完整度 70-90%**：建议补充关键功能
3. **完整度 < 70%**：需要继续开发
