# /run - 本地运行

## 描述

启动项目，验证功能是否正常运行。

## 前置条件

- **代码已实现**：存在 src/ 或其他代码目录

## 工作流程

```
用户输入 /run
  ↓
检测项目类型和启动命令
  ↓
执行启动命令
  ↓
验证核心功能
  ↓
报告运行结果
```

## 项目类型与启动命令

| 项目类型 | 启动命令 | 端口 |
|----------|----------|------|
| Next.js | `npm run dev` | 3000 |
| React (Vite) | `npm run dev` | 5173 |
| Node.js | `npm start` 或 `node src/index.js` | 3000 |
| Python/FastAPI | `uvicorn main:app --reload` | 8000 |
| Go | `go run main.go` | 8080 |

## 运行结果

| 结果 | 描述 | 后续操作 |
|------|------|----------|
| ✅ 成功 | 项目启动，功能正常 | 使用 /test 执行测试 |
| ⚠️ 警告 | 启动成功，但有警告 | 检查日志，评估影响 |
| ❌ 失败 | 启动失败 | 查看错误日志，修复代码 |

## 相关指令

| 指令 | 描述 | 关系 |
|------|------|------|
| `/dev` | 代码开发 | 前置步骤 |
| `/test` | 单元测试 | 下一步 |
| `/review` | 代码审查 | 后续步骤 |

## 常见问题

### 端口被占用

```bash
# macOS/Linux
lsof -ti:3000 | xargs kill -9

# Windows
netstat -ano | findstr :3000
taskkill /PID <pid> /F
```

### 依赖未安装

```bash
npm install
# 或
yarn install
```

### 环境变量缺失

检查 `.env.local` 或 `.env` 文件，确保所有必需的环境变量已配置。

## 后续操作

运行成功后，建议：
1. 测试核心功能是否正常
2. 使用 `/test` 执行单元测试
3. 使用 `/review` 进行代码审查
